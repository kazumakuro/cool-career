# アプリケーション要件定義書

## プロジェクト概要
- **プロジェクト名**: Cool Career - MBTIベースキャリアジャーニーアプリ
- **作成日**: 2025-07-29
- **更新日**: 2025-07-29
- **バージョン**: 1.0.0
- **技術スタック**: Next.js 15, TypeScript, Tailwind CSS, Supabase
- **プロジェクトタイプ**: Progressive Web App (PWA)

## 目的・ビジョン
### ミッション
「就職をスタートに、自分なりのキャリア哲学を持って長い社会人生を歩める人を増やす」

### ビジョン
全ての人が自分らしいキャリアを築き、仕事を通じて自己実現できる社会の実現

### 背景
- 500人以上の学生面談から得た「就職がゴールではない」という現場の気づき
- 既存の適職診断の課題：表面的な職種マッチングに留まり、「なぜ働くか」の本質的な問いに答えていない
- Z世代のMBTI人気を活用しつつ、より深い価値提供を目指す

### 差別化ポイント
1. **現場データに基づく信頼性**: 500人の実際の面談データから抽出した悩みパターン
2. **哲学的アプローチ**: 単なる適職診断を超えた「働く意味」への深い洞察
3. **継続的な成長支援**: 一回きりの診断ではなく、キャリアの節目での振り返り機能
4. **実用性とエンタメ性の両立**: SNSでバズる要素と実践的なキャリアアドバイスの融合

## ターゲットユーザー
### プライマリターゲット
1. **就職活動中の大学生・大学院生（約400万人）**
   - ペインポイント：自己分析の難しさ、適職がわからない不安
   - ニーズ：具体的な職種提案、自己PRの材料

2. **入社1-3年目の若手社会人（約300万人）**
   - ペインポイント：仕事への違和感、早期離職の誘惑
   - ニーズ：キャリアの方向性確認、転職判断の基準

3. **キャリア転換期の25-35歳（約500万人）**
   - ペインポイント：現状への疑問、将来への不安
   - ニーズ：キャリアチェンジの可能性探索

### セカンダリターゲット
- 部下を持つマネージャー（チームビルディング、1on1活用）
- 採用・人事担当者（カルチャーフィット判定）
- キャリアカウンセラー（相談ツールとして活用）

## 主な機能要件

### コア機能（MVP必須）

#### 1. スマートキャリア診断
- **概要**: 5分で完了する性格・価値観診断
- **診断内容**:
  - MBTI風16タイプ分類（50問以内）
  - ビッグファイブ要素の統合
  - 独自指標「社畜化リスクスコア」
- **出力**:
  - 性格タイプとその詳細解説
  - 向いている職種TOP5（理由付き）
  - 避けるべき環境・上司タイプ
  - 成長のためのアドバイス

#### 2. リアル哲学データベース
- **概要**: 実際に働く人々の「なぜこの仕事をしているか」を集めたデータベース
- **初期データ**: 100人分（MBTIタイプ×年代×職種で分類）
- **コンテンツ形式**:
  - 3分動画インタビュー
  - 音声メッセージ（ポッドキャスト風）
  - テキストストーリー
- **機能**:
  - タグ検索（#INTJ #営業 #30代 など）
  - お気に入り保存
  - 類似度マッチング

#### 3. パーソナルビジョンビルダー
- **概要**: 診断結果と哲学DBを基にした個人のキャリアビジョン作成
- **作成フロー**:
  1. 診断結果の振り返り
  2. 共感した哲学の選択
  3. 自分の価値観の言語化
  4. 5年後のビジョン設定
- **出力**:
  - 「私のキャリア哲学」ステートメント（A4 1枚）
  - 行動指針3箇条
  - 定期見直しスケジュール

### サブ機能（段階的実装）

#### Phase 1: バズ＆エンゲージメント機能（リリース後1ヶ月）
1. **社畜化リスク診断**
   - カウントダウンタイマー表示
   - 予防アクション提案
   - SNSシェア機能

2. **隠れ天職発見**
   - 意外性のある職種提案アルゴリズム
   - 転職成功者の実例紹介
   - 「もしも」シミュレーション

3. **地雷上司診断**
   - 相性の悪い上司タイプ判定
   - 面接での見極めポイント
   - 対処法アドバイス

#### Phase 2: 継続利用促進機能（3ヶ月後）
1. **キャリア成長トラッカー**
   - 3ヶ月ごとの定期診断
   - 価値観の変化可視化
   - 成長グラフ表示

2. **キャリアシミュレーター**
   - 22歳→65歳のビジュアルノベル
   - 選択による分岐
   - 満足度・後悔度スコア

#### Phase 3: 収益化機能（6ヶ月後）
1. **プレミアムメンタリング**
   - AI相談チャット（24時間対応）
   - 同じMBTIタイプの先輩マッチング
   - 1on1セッション予約

2. **企業向けサービス**
   - 採用時のカルチャーフィット診断
   - チーム相性分析
   - 離職予防プログラム

## 非機能要件

### パフォーマンス
- **レスポンスタイム**: 
  - 診断画面の遷移: 1秒以内
  - 結果表示: 3秒以内
  - API応答: 500ms以内
- **同時接続数**: 1,000ユーザー
- **診断完了率**: 80%以上を目標
- **可用性**: 99.5%（月間ダウンタイム約3.6時間以内）

### セキュリティ
- **認証**: Supabase Auth（OAuth対応）
- **データ暗号化**: 
  - 通信: HTTPS/TLS 1.3
  - 保存: AES-256
- **個人情報保護**:
  - GDPR準拠
  - 診断データの匿名化
  - 最小権限の原則
- **セッション管理**: JWT（有効期限24時間）

### ユーザビリティ
- **デザイン原則**:
  - モバイルファースト（スマホ利用率70%想定）
  - ダークモード対応
  - アクセシビリティ（WCAG 2.1 AA準拠）
- **対応環境**:
  - ブラウザ: Chrome, Safari, Firefox, Edge（各最新2バージョン）
  - デバイス: iOS 14+, Android 10+
- **言語**: 日本語（将来的に英語対応）

### 信頼性
- **データバックアップ**: 日次自動バックアップ
- **エラー処理**: ユーザーフレンドリーなエラーメッセージ
- **監視**: エラーログ、パフォーマンスメトリクス

## データモデル（主要エンティティ）

### User
- id (UUID)
- email (unique)
- created_at
- updated_at
- profile (JSON): 年齢、性別、職業など

### DiagnosisResult  
- id (UUID)
- user_id (FK)
- mbti_type (VARCHAR)
- big_five_scores (JSON)
- risk_score (INTEGER): 社畜化リスク
- recommended_jobs (JSON[])
- created_at

### Philosophy
- id (UUID)
- contributor_id (FK)
- mbti_type (VARCHAR)
- age_group (VARCHAR)
- job_category (VARCHAR)
- content_type (ENUM): video/audio/text
- content_url (TEXT)
- tags (TEXT[])
- likes_count (INTEGER)

### CareerVision
- id (UUID)
- user_id (FK)
- statement (TEXT)
- action_guidelines (JSON[])
- inspired_philosophies (UUID[]): 参考にした哲学
- created_at
- updated_at

### UserActivity
- id (UUID)
- user_id (FK)
- action_type (VARCHAR)
- timestamp
- metadata (JSON)

## 画面設計

### 主要画面一覧
1. **ランディングページ** (`/`)
   - ヒーローセクション
   - 診断開始CTA
   - 機能紹介

2. **診断フロー** (`/diagnosis`)
   - 質問画面（プログレスバー付き）
   - 診断中アニメーション

3. **診断結果** (`/result/[id]`)
   - MBTIタイプ表示
   - 社畜化リスクメーター
   - 推奨職種カード
   - シェアボタン

4. **哲学データベース** (`/philosophies`)
   - フィルター（タイプ/年代/職種）
   - カード型リスト表示
   - 詳細モーダル

5. **ビジョンビルダー** (`/vision`)
   - ステップ型フォーム
   - プレビュー機能
   - PDF出力

6. **マイページ** (`/mypage`)
   - 診断履歴
   - 保存した哲学
   - ビジョン管理

### 画面遷移フロー
```
ランディング → 診断フロー → 結果表示
                              ↓
                          哲学DB閲覧
                              ↓
                        ビジョン作成
                              ↓
                          マイページ
```

## API設計（主要エンドポイント）

### 診断関連
- `POST /api/diagnosis/start` - 診断開始
- `POST /api/diagnosis/answer` - 回答送信
- `GET /api/diagnosis/result/:id` - 結果取得

### 哲学DB関連
- `GET /api/philosophies` - 一覧取得（フィルタリング対応）
- `GET /api/philosophies/:id` - 詳細取得
- `POST /api/philosophies/:id/like` - いいね

### ビジョン関連
- `POST /api/vision` - ビジョン作成
- `PUT /api/vision/:id` - ビジョン更新
- `GET /api/vision/:id/pdf` - PDF生成

### ユーザー関連
- `GET /api/user/profile` - プロフィール取得
- `PUT /api/user/profile` - プロフィール更新
- `GET /api/user/history` - 履歴取得
